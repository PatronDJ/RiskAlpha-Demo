Executive summary
The problem — and the solution

Labassa Capital currently operates a $600M project finance book across fragmented spreadsheets, manual reporting processes and disconnected systems. As the fund grows, this creates compounding risks: reporting errors, compliance gaps, operational bottlenecks and an inability to scale. The proposed platform eliminates these by replacing every manual process with a connected, automated and auditable digital system.

Current state — the pain points

1
Investor reporting is manual and error-proneMonthly statements, NAV calculations and distribution notices are assembled by hand from Excel workbooks. Each reporting cycle takes days and carries significant operational risk.
2
Loan calculations live in disconnected spreadsheetsAccruals, waterfall distributions and covenant tracking are maintained in separate files with no single source of truth. Reconciliation is manual and time-consuming.
3
KYC and onboarding is entirely paper-basedInvestor onboarding relies on physical documents, manual ID checks and email chains. There is no digital audit trail for AML/CTF obligations.
4
ASIC compliance is reactive, not automatedRG45 and RG133 disclosures require significant manual data gathering each quarter. There is no immutable audit trail to support regulatory examination.
5
No scalable knowledge or SOP infrastructureOperational procedures exist in individuals' heads or informal documents. There is no version-controlled SOP library, intranet or team knowledge base.

Future state — the outcomes

Zero manual reporting
Investor statements, NAV calculations and distribution notices generated automatically from a locked data warehouse snapshot. Period close takes hours, not days.
Single source of truth
Every loan balance, accrual and distribution calculation traces back to one auditable data warehouse table. No more reconciling spreadsheets.
Digital investor experience
MFA-secured portal where investors self-serve statements, holdings and fund documents — with automated email delivery on each reporting event.
ASIC-ready by design
RG45 and RG133 data points calculated automatically from the warehouse. Immutable audit log covering every calculation and approval decision.
Scalable to $1B+
Cloud-native infrastructure on Snowflake + Azure scales linearly with book size. Adding a new loan or investor requires no additional manual process.
Platform built to last
All code version-controlled in Git, all processes documented in Confluence. New team members can onboard in days, not months.

Build vs buy — the key decision for Labassa at $600M
Buy/Build — Depending on the number of loans
These handle the operational side — drawdowns, repayments, facility amendments. They become the source of truth that feeds the warehouse via API or SFTP. Enterprise platforms like Loan IQ or nCino are bank-grade overkill at this scale.
Build — calculation engine
Python (pandas / numpy / scipy) runs outside Snowflake on an Airflow worker or Azure Function. It does the heavy calculation work, reads raw data from Snowflake and writes results back as clean tables. dbt then handles SQL-layer transformations inside Snowflake. All logic is version-controlled in Git with pytest unit tests — fully auditable.

Period close workflow — lock, review and approve before any statement is generated
1
Period lock
Month/quarter end reached. Calc engine writes final snapshot to Snowflake with is_period_end = true. No further source edits allowed.
2
Auto reconciliation
dbt tests run — bank rec, loan balance, assets = liabilities + equity. Any failure blocks report generation automatically.
3
4-eyes approval
Fund manager reviews NAV in approval portal. Signs off — approved_by and approved_at written. is_locked = true. Audit trail complete.
4
Report generation
Airflow triggers document pipeline. Templates populated from approved snapshot. PDFs stored in Azure Blob. Email delivery queued.
